# MVP Техническое задание - Октябрь 2025

**Дата:** 8 октября 2025  
**Срок выполнения:** 31 октября 2025  
**Цель:** Создание минимально жизнеспособного продукта (MVP) платформы предсказательных рынков  

---

## 🎯 Цели MVP

### Основные задачи
1. **Создание базового функционала** торговли на предсказательных рынках
2. **Интеграция с блокчейн-инфраструктурой** (Masterchain для России)
3. **Реализация системы оракулов** для разрешения рынков
4. **Обеспечение безопасности** и базовой защиты от мошенничества
5. **Подготовка к масштабированию** до 1000+ пользователей

### Ключевые метрики успеха
- **1000+ зарегистрированных пользователей**
- **50+ активных рынков**
- **$10K+ общий объем торгов (GMV)**
- **99%+ uptime**
- **<100ms latency** для основных операций

---

## 👥 Команда разработки

### Blockchain Engineer
**Основные задачи:**
- Разработка смарт-контрактов для создания и управления рынками
- Реализация системы эскроу для безопасного хранения средств
- Интеграция с Masterchain и подготовка к работе с цифровым рублем
- Создание механизмов разрешения рынков через оракулы

**Технические требования:**
- Опыт работы с Solidity и Ethereum-совместимыми сетями
- Знание архитектуры Masterchain
- Понимание принципов работы оракулов и механизмов консенсуса
- Опыт аудита смарт-контрактов

**Deliverables:**
- Смарт-контракт `MarketFactory.sol` для создания рынков
- Смарт-контракт `Escrow.sol` для управления средствами
- Смарт-контракт `Oracle.sol` для разрешения рынков
- Документация по развертыванию и тестированию

### Frontend Engineer
**Основные задачи:**
- Создание пользовательского интерфейса для торговли
- Реализация интеграции с кошельками (MetaMask, WalletConnect)
- Разработка мобильной версии (PWA)
- Создание административной панели

**Технические требования:**
- React 18+ с TypeScript
- Опыт работы с Web3 библиотеками (ethers.js, wagmi)
- Знание Tailwind CSS для стилизации
- Опыт создания PWA приложений

**Deliverables:**
- Главная страница с лентой рынков
- Страница торговли с графиками и ордербуком
- Личный кабинет пользователя
- Административная панель для управления рынками
- Мобильная версия (PWA)

### Backend Engineer
**Основные задачи:**
- Разработка API для взаимодействия с блокчейном
- Создание системы управления пользователями
- Реализация торгового движка (CLOB)
- Интеграция с внешними источниками данных

**Технические требования:**
- Node.js с Express/FastAPI
- PostgreSQL для хранения данных
- Redis для кэширования
- Опыт работы с Web3 и блокчейн API

**Deliverables:**
- REST API для всех операций платформы
- Система аутентификации и авторизации
- Торговый движок с поддержкой лимитных ордеров
- Интеграция с оракулами (ЦИК, MOEX, Росстат)
- Система уведомлений

## 📋 Функциональные требования

### 1. Управление пользователями
- **Регистрация** через email/телефон
- **Верификация** номера телефона
- **Подключение кошелька** (MetaMask, WalletConnect)
- **KYC процедуры** (базовый уровень)
- **Управление профилем**

### 2. Создание и управление рынками
- **Создание бинарных рынков** (Да/Нет)
- **Настройка параметров** (дата разрешения, описание)
- **Привязка к оракулам** (внешние источники данных)
- **Модерация контента** администраторами

### 3. Торговая система
- **Размещение ордеров** (покупка/продажа)
- **Лимитные ордера** с указанием цены
- **Отмена ордеров** до исполнения
- **Просмотр истории** сделок
- **Мониторинг портфеля**

### 4. Система оракулов
- **Интеграция с ЦИК** (результаты выборов)
- **Интеграция с MOEX** (финансовые данные)
- **Интеграция с Росстат** (экономические показатели)
- **Автоматическое разрешение** рынков
- **Ручное разрешение** при спорных ситуациях

### 5. Безопасность
- **Защита от ботов** (device fingerprinting)
- **Rate limiting** для API
- **Валидация входных данных**
- **Логирование** всех операций
- **Мониторинг** подозрительной активности

## 🔧 Технические требования

### Производительность
- **Latency:** <100ms для основных операций
- **Throughput:** 1000+ запросов/секунду
- **Uptime:** 99%+ доступность
- **Concurrent users:** 1000+ одновременных пользователей

### Безопасность
- **HTTPS** для всех соединений
- **JWT токены** для аутентификации
- **Валидация** всех входных данных
- **Защита от SQL injection** и XSS
- **Регулярные бэкапы** данных

### Масштабируемость
- **Горизонтальное масштабирование** backend сервисов
- **Кэширование** часто запрашиваемых данных
- **CDN** для статических ресурсов
- **Load balancing** для распределения нагрузки

## 📊 Интеграции

### Внешние API
- **ЦИК РФ** - результаты выборов и референдумов
- **Московская биржа (MOEX)** - котировки и индексы
- **Росстат** - экономические показатели
- **ЦБ РФ** - ключевая ставка и курсы валют

### Блокчейн интеграции
- **Masterchain** - основная сеть для России
- **MetaMask** - подключение кошельков
- **WalletConnect** - мобильные кошельки
- **Web3.js/Ethers.js** - взаимодействие с блокчейном

---

## 🧪 Тестирование

### Типы тестов
- **Unit тесты** - покрытие 80%+ кода
- **Integration тесты** - тестирование API
- **E2E тесты** - полные пользовательские сценарии
- **Load тесты** - проверка производительности
- **Security тесты** - проверка уязвимостей

### Тестовые сценарии
1. **Регистрация и верификация** пользователя
2. **Создание рынка** администратором
3. **Размещение ордера** пользователем
4. **Исполнение сделки** при совпадении ордеров
5. **Разрешение рынка** через оракул
6. **Выплата выигрышей** победителям
